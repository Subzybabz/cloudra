01 · Capabilities

Four pillars
of migration intelligence.

Each assessment runs through a rigorous pipeline — from real-time cost estimation to automated compliance documentation. No guesswork, no spreadsheets.

— 01

Dynamic Risk Scoring

Fourteen migration parameters are processed through piecewise normalisation functions and weighted sub-score calculations across three dimensions. The composite risk score maps to actionable LOW, MEDIUM, or HIGH risk tiers with automatic mitigation recommendations.

Piecewise Normalisation Weighted Sub-Scores
— 02

Live AWS Pricing

Real-time EC2 instance pricing is queried directly from the AWS Pricing API via Boto3, with a strict 1.5-second timeout. If internet or credentials are unavailable, the system falls back seamlessly to cached pricing tables — no disruption, no errors.

Boto3 Integration Offline Fallback
— 03

Audit Trail Database

Every assessment is persisted in an SQLite database with a schema aligned to DynamoDB design patterns. Full history, live search, record retrieval, and soft deletion — all accessible from the dashboard sidebar. Your audit trail is always intact.

SQLite Persistence DynamoDB-Aligned Schema
— 04

PDF Compliance Reports

Generate print-ready compliance audit PDFs on the fly. Each report includes input parameters, composite and dimensional risk scores, pricing estimates, and prioritized mitigation recommendations — formatted with ReportLab for professional delivery.

ReportLab Generation One-Click Download
02 · Workflow

Three steps
to clarity.

01

Configure Parameters

Enter your migration context — data volume, server count, application complexity, AWS region, security posture, and compliance scope. Fourteen parameters across three dimensions.

02

Calculate Risk Profile

The scoring engine normalises each input, computes weighted dimensional sub-scores, and produces a composite risk tier with real-time cost estimates from AWS.

03

Review & Export

Visualize results through gauge and radar charts, review prioritized mitigation recommendations, and download a comprehensive PDF audit report for stakeholder review.

03 · Risk Dimensions

Three lenses
on your migration.

Each dimension carries independent weight in the composite score, ensuring no single factor dominates the risk profile.

35%

Operational

O = 0.30·ND + 0.25·NS + 0.30·NA + 0.15·NW

Data volume, server count, application complexity, and migration window duration — the physical realities of moving infrastructure at scale.

30%

Financial

F = 0.45·NC + 0.30·NE + 0.25·NT

Monthly cloud cost estimates against on-premises CapEx, network egress charges, and projected user scaling — the economic calculus of migration.

35%

Cybersecurity

Cy = 0.35·NSens + 0.30·NIAM + 0.20·NEnc + 0.15·NComp

Data sensitivity classification, IAM permissiveness posture, encryption coverage, and compliance governance scope — the security surface of your migration.
